Programme Grid pages

Clicking on one of the grid-times buttons on the main page will display the programme grid for that given day and time-chunk.

Above the grid

At the very top of the page, there's the usual banner with shortcuts to the Main page, Grids, People, Items, Rooms, etc.

Beneath the banner, it'll say the day and time period "e.g. Friday 2pm-6pm", which also appears in the title of the browser's window or tab.

Next, there are several arrow buttons. The left and right arrow buttons (when present) move you through earlier and later in the day, respectively. The up and down arrow buttons move you to the same time on the previous and later days, respectively. Be aware that the grid's orientation for time can vary, so the arrow arrangement might not always make sense to you.

These arrows also appear beneath the grid, too.

The grid

The grid shows a table of rooms vs time-slots. The orientation can vary.

By default, for Redemption, grid lists the Rooms down the Y axis, and time along the X axis, in the manner of Redemption's Programme Booklet, while for Odyssey, the X and Y axes are flipped, so that you have rooms along the top, and times down the side. This is Judith's preference.

Rooms are listed by their grid order (see Rooms); this is more convenient than, say, alphabetical, because it means the most commonly-used rooms are the first you see.

Rooms should only be listed if they have "show in grid" set for them. So, for example, Secure Store and Ops rooms shouldn't show up here. However, at time of writing, I don't think I've implemented that - I think it'll show all rooms.

Initially, the database will be devoid of programme items, and so you won't see a very exciting grid: every slot will contain a "Fill Slot" button, so it might be better to come back to this page after you've entered a few programme items...

Each slot on the grid will show one of two things:

  • If there are programme items assigned to this room at this time, they'll be listed. There may be multiple item using the room at any one time (if you're partway through rearranging things), so all will be shown.
  • Otherwise, you'll have a "Fill slot" button, which allows you to assign already-existing items to this time slot.

Programme grids can display items in lots of modes, controlled by the switches between the grid (and described further down this page), but generally you might see:

  • The item's title (the bit in bold).
  • The names of anyone associated with the item. "(no-one)", if there isn't any.
  • Any tags associated with the item.
On the Redemption database, you'll also see items' shortnames. You can ignore them on the Odyssey database, because they're all auto-generated and don't contain useful information.

In each case, all the things displayed in a given room/time square are links:

  • Clicking on the item's title (or shortname) takes you to the item's page.
  • Clicking on a person's name takes you to the person's page.
  • Clicking on the item's tags (name or image) takes you to the tags's page.

Note that if an item name or a person name is in italics, then it indicates that that item/person is marked as "don't show in programme booklet".

Fill Slot

When you click on "Fill Slot", you're presented with a new page, entitled "Move what to this slot?". The page has a drop-down list to allow you to select the programme item you want to move . The list shows all the items, sorted by title (or shortname, for Redemption), but it lists items without real rooms (i.e. those in Nowhere, or that really don't have a room) first, and then it lists the rooms which already have a real room. You can tell the difference because the items with real rooms are prefixed with "-".

When you've found the item you want to position there, you click the "Move" button, and the programme item will be moved.

Somewhat annoying, you're then presented with a page that says the items been moved, and which presents you with the option to view the item (the link) or go back to the grid you were on (the "Back to grid" button). You probably just want to be taken back to the grid you were on, without being asked.

At time of writing, there isn't a way of saying, "create new item here", which both Steve R and Judith have asked for. With the exception of the Opening and Closing ceremonies, I think it's unlikely you're going to know, on first entry, which room and what time a programme item will have, which is why the database currently works the way it does, but this can be changed.

Grid options

At the bottom of the page, there are a load of choices for controlling how the grid is displayed. Choose how you'd like to see things, then click Redraw.

  • Shortnames vs titles
    • Show Shortnames - Display shortnames, but not titles. Only useful on Redemption.
    • Show Titles - Display titles, but not shortnames. Default on Odyssey.
    • Show Shortnames - Display shortnames, but not titles. Only useful on Redemption.
    • Shortname:Title - Display both, colon-separated. only useful on Redemption.
    • Neither - Display neither title nor shortname.
  • Orientation
    • Time: left to right - Time along the top (X axis), Rooms down the side (Y axis).
    • Time: top to bottom - Rooms along the top (X axis), Time down the side (Y axis).
  • Real names vs Badge names
    • Joe Bloggs - Display real names
    • Joey - Display Badge names (if any - real names otherwise)
    • Joe Bloggs [Joey] - Display real names, followed by badge name if any.
    • Joey (Joe Bloggs) - Display badge names, if any, followed by real names
    • Neither - Don't display people associated with items.
  • Tags: selector
    • No tags - Don't display any tags (regardless of format).
    • All tags - Display all tags (in selected format).
    • Specific tag - Display only those items tagged with the selected tag
  • Tags: format
    • Tag image - Just display tag images (ignored if No Tags)
    • Tag name - Just display tag names (ignored if No Tags).
    • No tags - Don't display tags.
    • Both - Display both tag names and tag images (ignored if No tags).
