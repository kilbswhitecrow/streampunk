Main page

The main page can be found here.

It has two main components: the grids, and "Other Tools".

Grids

The grids are explained in more detail elsewhere, but the basic idea is:

  • There's a row for each day of the con, including non-public days.
  • Each row is split into individual chunks of time (four hours in length, by default). Clicking on one of those chunks will show you a programme grid for that period of time.
  • At the far right of each row, there's a button for a grid for the whole day.
  • Beneath the table of grids, there's a button for "Entire grid". That'll list your entire programme in one go.
You can use the grid pages to move existing items around. Once the database has a large number of items, you'll probably use this a lot.

Other Tools

This part of the page gives you links to the pages where you can access, create and generally fiddle around with the contents of the database.

The first part is concerned with the main kinds of data in the database: people, items, rooms, tags and tech. For each of them, you've got three or four choices:

  • List - list the people/items/whatever that are already entered (which gives you links to the editing features).
  • Create - Create a new person/item/whatever from scratch.
  • Delete - Delete one of the existing people, items, rooms, etc. Rare that you'll need this.
  • Availability - Set up or alter the availability of something. This is explained in more detail elsewhere, but the basic idea is that, usually people, rooms and tech are available for the entire con, but this might not always be the case, so where necessary, you can configure the periods when something is available - then, later, you can run a check to ensure that it/he/she is not required outside those periods.

The next part - items/people, items/rooms, items/tech - is where you can run basic checks, or change the relationships between items and people, rooms or tech.

  • Items/People
    • (missing people) - Will show you items that don't have anyone attached.
    • People clashes - Will show you where the same person is attached to more than one item at the same time. Also shows when someone isn't available for a programme item.
    • (by person) -Lists programme items, by person (all Bob's items, then all Carol's, and so on).
    • (by item) -Lists people, by programme item (all the people on item X, then all the people on item Y, and so on.).
    • (people by item count) -Lists people, giving the number of programme items to which they're attached. The list is sorted so that the most heavily-scheduled people are listed first.
    • (programmed people not joined) -Lists people who are attached to programme items, but for which the database doesn't yet have a membership number (i.e. their membership number is -1). It's possible (even likely) that they've joined, but their membership number hasn't been added to their entry in the programme database.
    • Add person to item - Allows you to add a person to an existing item. You can also do this directly from the item.
  • Items/rooms
    • (missing a room) - Will show you all the items that aren't currently allocated to a given room, and will tell you how many items are allocated to Nowhere, the pseudo-room.
    • Room clashes - Will show you where more than one item occupies the same room during the same time slot.
    • Change an item's room - Move an item from one room to another. Easiest done on the grids.
    • Change an item's start day/time - Allows you to specify a new day and start time for an item. Easiest done on the grids.
  • Items/tech
    • Tech clashes - Will show you any cases where the same item of tech is assigned to two programme items at the same time.

The final part - Manage - allows you to fiddle with the bits and pieces that tie everything together, but which don't normally need to change.

  • Seating Layouts - Are how rooms are laid out, e.g. theatre style, cabaret, boardroom. Used when configuring room capacities, and for specifying how a given programme item's room should be laid out. Here's where you'd go to add another one - for example, you might want to add a new layout, for "chairs in a circle".
  • Item kinds - Things like Panel, Quiz, Talk, Workshop, and so on. You might want to add "Tour", "Training", "Setup", etc.
  • Tech classes - For some kinds of tech, you'll have numerous instances of them, and they're assigned classes. Flip-charts, microphones, laptops, etc.
  • Tech sources - Are where you get the tech items from. Stage Electrics might be one source, and Chris O'Shea or Eddie Cochrane might be a couple more.
  • Tech basis - Currently includes Borrow, Hire and Own
  • Departments - Who the item of tech is assigned to. Ops, Stewards, Green Room, etc.
  • People roles - When a person is associated with a programme item, they do so in a particular role, such as Moderator, Participant, Quizmaster, Scorer, Gopher.
  • People statuses - Currently includes Possible, Pending and Confirmed, and indicates how solid the person's association with the item is, at present.

When you modify any of these categories, you can also specify which one is the default. Generally, Judith has the default set to "TBA".
